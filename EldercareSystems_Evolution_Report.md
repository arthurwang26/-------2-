# 基於多模態與大語言模型之高齡照護行為分析系統演進
**(Evolution of Multimodal LLM-based Eldercare Action Analysis Systems)**

## 摘要 (Abstract)
本報告詳細梳理了「高齡照護行為分析系統」從初代 (System 1) 到第三代 (System 3) 的演進歷程。整個演進過程旨在解決實際長照監視器場景中遇到的嚴苛挑戰，包含：運算資源瓶頸、主觀情緒導致的大模型幻覺、連續長動作的遺漏、以及臉部辨識在遮蔽環境下的污染問題。第三代系統透過「多數決防毒」、「滑動視窗時序分析」與「物理邏輯校正」等創新機制，達成了極高的精準度與穩定性。

---

## ▍第一代系統 (Eldercare System 1)：全模組化堆疊與概念驗證
**定位**：系統的概念驗證 (PoC)，採用最前沿的多模態 AI 模型進行全方位特徵提取。

### 核心架構
1. **物件與軌跡追蹤**：使用 `YOLOv8` 偵測畫面中的人物與日常物件，並結合 `ByteTrack` 進行多目標軌跡追蹤 (MOT)。
2. **動作辨識**：雙軌並行，同時使用 `VideoMAE` (提取 RGB 影像時空特徵) 與 `RTMPose` (提取人體骨架)。
3. **情緒分析**：導入 `HSEmotion`，分析畫面上人物的臉部表情特徵以判斷情緒狀態。
4. **LLM 總結**：將所有偵測到的事件寫成 Prompt，交由雲端大語言模型生成照護報告。

### 遭遇痛點與瓶頸
- **主觀幻覺嚴重**：監視器畫面通常模糊且視角偏高，`HSEmotion` 極易誤判情緒。而大模型在接收到這些錯誤的情緒提示後，會自行腦補出「長輩心情低落」等未經證實的主觀情節，失去客觀性。
- **運算資源耗竭**：`VideoMAE` 是重量級的 Transformer 模型，在邊緣運算設備上處理長影片會導致嚴重的記憶體 (VRAM) 溢出。
- **動作單一化**：影片只能對應單一動作標籤，無法捕捉時間序列上的行為改變。

## ▍第二代系統 (Eldercare System 2)：解耦、輕量化與物理客觀性確立
**定位**：確立「物理客觀性優先」的開發原則，大幅降低運算成本，為商業化與落地佈署鋪路。

### 核心架構演進
1. **情緒辨識全面拔除**：強制要求 LLM 與所有視覺模組「絕對禁止描述或推測心理感受，僅專注於物理動作」。
2. **捨棄 VideoMAE，擁抱輕量骨架模型**：全面改用**客製化的 Transformer + Bi-LSTM 骨架動作辨識模型**。該模型僅依賴 17 關節點的空間特徵與時間差 (Velocity)，特徵維度僅 102 維，運算速度極快且不易受背景光源干擾。
3. **引入 Zero-shot HOI (人機互動)**：利用輕量級 `CLIP` 模型搭配邊界框裁切，詢問人物與物品之間的關聯（例如 Drinking_From、Sitting_On），並額外加上 `BLIP` 作為 VQA (視覺問答) 雙重驗證，避免 CLIP 的零樣本幻想。

### 遭遇痛點與瓶頸
- **ReID (身份重識別) 污染**：追蹤系統一開始雖使用 `InsightFace` 比對家屬（如：陳爺爺、王奶奶），但只要軌跡中有一幀因為角度問題發生誤判，系統就會把這個錯誤身份套用到後續所有動作上，導致張冠李戴。
- **動作截斷問題 (Truncation)**：LSTM 模型的設計是輸入 60 幀 (約 2 秒) 的時間序列。若影片長達 10 秒，系統只會取前 2 秒推論，導致「先走路、後坐下」的複雜行為中，只有最初的動作被記錄下來。

## ▍第三代系統 (Eldercare System 3)：時序平滑、防毒與邏輯對齊 (巔峰之作)
**定位**：針對演算法邊界情況 (Corner cases) 進行極致打磨，賦予系統時間解析度與自我校正能力。

### 核心架構演進與創新機制
#### 1. 臉部身份防毒機制 (Majority Voting ReID)
為了解決 ReID 污染，System 3 放棄了「一幀定生死」的邏輯。系統會先讓追蹤器盲追 (Blind Tracking)，收集該人物在整段影片中所有的臉部 Embedding，並與資料庫比對。最後以**「最高票多數決 (Canonical Identity)」**的方式，為這條軌跡賦予最終身份。只要軌跡中正確的臉部辨識佔多數，極少數的誤判雜訊將被徹底淹沒。

#### 2. 骨架尺度正規化 (Scale Normalization)
解決監視器畫面中「近大遠小」的誤差。將擷取到的絕對像素座標，依據畫面寬高嚴格映射至 `[0, 1]` 的相對空間中，確保模型對距離攝影機不同遠近的長輩，都能給出一致的判定標準。

#### 3. 滑動視窗推論 (Sliding Window Action Parsing)
賦予模型「時間解析度」。捨棄一次性推論，改以 **Window=60 幀、Stride=30 幀** 的步幅，如同掃描機般掃過整段軌跡資料。這使得系統能夠在一支影片中，連續輸出 `[0~2秒: 走路] -> [2~4秒: 坐下]` 的複合事件陣列。

#### 4. 時序平滑化與物理邏輯修正 (Temporal Smoothing & Logic Correction)
由於滑動視窗在動作切換邊界容易產生雜訊，系統實裝了強大的後處理機制：
- **初始孤立雜訊消除**：YOLO 在前幾幀剛鎖定目標時，位移特徵不明顯，常把走路開頭誤判為 `Standing`。系統會自動將極短暫的起步 `Standing` 吸收到後續的 `Walking` 之中。
- **夾心雜訊抹平**：若在連續的相同動作（如 `Walking`）中夾雜不到 2 秒的異樣判定，將視為干擾予以抹平。
- **物理定律反轉校正**：當連續動作模型產生延遲，輸出違背現實物理定律的 `Sitting -> Sit down` (先坐著再坐下) 或 `Standing -> Stand up` 時，系統會強制作出時序對調校正，還原為正確的 `Sit down -> Sitting`。

#### 5. SmolVLM2 全局語境與大模型 (Qwen3) 整合
- **全局語境捕捉**：每 30 幀呼叫 `SmolVLM2` 進行截圖描述（例如：「兩位老人坐在客廳桌前交談」），彌補純物理特徵缺乏環境背景的缺點。
- **知識圖譜匯出**：所有 Action 與 HOI 互動特徵會被匯出為 Markdown 與 Neo4j Cypher 格式的知識圖譜，可供圖資料庫進行長期行為挖掘。
- **純客觀報告生成**：透過 `llama-cpp` 載入 `Qwen3-4B`，將上述所有結構化且經由雙重驗證的物理特徵撰寫成人類可讀的中文報告，徹底達成「精準追蹤、行為解構、客觀描述」的三位一體。

---

## 結論
從 System 1 到 System 3 的演化，展現了如何將最先進的 AI 模型從「能跑」優化到「實用」的過程。透過模組的解耦、多數決追蹤演算法的導入，以及滑動視窗搭配物理邏輯平滑化的創新，Eldercare System 3 已經成為一套能克服監視器死角、過濾模型幻覺，並精準還原長輩連續行為軌跡的高階監護引擎。
