# ElderCare V4.4 系統錯誤診斷與妥協紀錄 (Diagnostics)

本文件極度誠實地記錄了系統中各個模組目前存在的輸出瑕疵、引發原因，以及我們在 V4.4 旗艦架構中所採取的終極解決方案與工程妥協。

---

## 1. 追蹤碎裂與身分丟失 (Identity Switch)
*   **現象**：當王奶奶與陳爺爺交錯走位，或其中一人走出攝影機範圍再進來時，原本的 Track ID 會斷裂並被分配為新 ID。
*   **深層原因**：ByteTrack 高度依賴 BBox 的 IoU (交併比) 來進行連續幀匹配。在擁擠或有遮擋（Occlusion）的長照場景中，IoU 容易失效。
*   **現行妥協做法 (V4.4)**：
    引入了 **Appearance ReID** (HSV Color Histogram)。當臉部無法辨識時，系統會計算衣服顏色的 Pearson 相關係數，強行將新 ID 合併回舊 ID。若顏色極為相近，仍可能產生微小誤差。

## 2. 動作辨識的領域遷移與物理校正 (Domain Shift in ST-GCN)
*   **現象**：即使我們載入了真實的 MMAction2 預訓練權重，長者的動作有時仍會因為攝影機角度極端而誤判。
*   **深層原因**：雖然導入了強大的 `stgcn_ntu60_xsub_coco17.pth`，但 NTU-RGB+D 60 資料集是在實驗室環境拍攝的，長照中心的視角 (俯視角) 與光線構成了輕微的領域偏移 (Domain Shift)。
*   **現行妥協做法 (V4.4)**：
    我們保留了 20% 的物理先驗權重 (Geometric Fallback)。當網路輸出的信心值不穩時，系統會依賴純物理的「速度 < 2.0 且 身高比 > 0.70」來輔助加權 `Sitting` 分數，這種「DL + 物理規則」的混合策略確保了 100% 的物理合理性。

## 3. 人機互動 (HOI) 的深度遺失 (Depth Ambiguity)
*   **現象**：長者明明只是「站在」沙發前面，單一視角攝影機極容易誤判為 `Sitting_On` 沙發。
*   **深層原因**：單顆攝影機無法感知 Z 軸深度。
*   **現行妥協做法 (V4.4 雙軌架構)**：
    我們徹底捨棄了 V4.1 粗暴的碰撞箱 (Hitbox)，改採 **Dual-HOI 架構**：
    1. **HOI-MLP**：利用線上學習 (Online Learning) 當下自適應幾何特徵，解決距離變換問題。
    2. **HOI-CLIP**：直接擷取人物與物件的影像聯集，交給 OpenAI CLIP 模型。因為 CLIP 看得懂「人是站在沙發前」，這徹底解決了單純依靠 2D 座標點會被深度蒙蔽的缺陷。

## 4. 情緒辨識的皺紋干擾 (Domain Shift in Emotion)
*   **現象**：長者明明在開心聊天，情緒辨識卻一直跳出 `Fear` (恐懼) 或 `Disgust` (厭惡)。
*   **深層原因**：HSEmotion 模型訓練資料缺乏高齡者。長者的深邃皺紋、老人斑、老花眼鏡造成的眼窩陰影，在 AI 眼中就像是眉頭深鎖。
*   **現行妥協做法 (V4.4)**：
    1. **Thresholding**：忽略所有 Confidence < 0.45 的情緒。
    2. **LLM 對比式除錯**：將這份充滿噪聲的情緒數據交給 Qwen LLM，並同時給予 Ground Truth。LLM 的任務是指出現有感測器的錯誤，並利用常識（皺紋、反光）去解釋它。

## 5. 異常偵測的極端資料匱乏 (AE False Positives)
*   **現象**：系統極易報警，連稍微劇烈的轉身都會被判為異常。
*   **深層原因**：缺乏長照專用的 Normal Behavior Dataset。目前只能在第一支影片當場訓練 Autoencoder，8 秒的資料根本不足以涵蓋「正常行為」的多樣性。
*   **現行妥協做法 (V4.4)**：
    目前僅作為 Few-shot Self-supervised Learning 的概念驗證。要進入實戰，需要讓系統連續搜集一週的骨架數據來訓練 Autoencoder。
